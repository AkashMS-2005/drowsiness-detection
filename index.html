Process

How It Works

Four intelligent steps to detect drowsiness in real-time and alert the driver before accidents occur.

01
Face Detection
Haar Cascade detects the driver's face from live webcam feed in every frame.
02
Eye Extraction
Eye regions are mathematically extracted from the face bounding box — no eye cascade needed.
03
CNN Prediction
Trained CNN model predicts open/closed state for each eye with 99% accuracy.
04
Alert System
Drowsiness score tracked over frames — audio alert fires when threshold exceeded.
Detection States

Alert System

Three distinct states monitored continuously in real-time.

👁️
AWAKE
Eyes open detected. Driver is alert. Score decreasing. Green indicator active.
😪
DROWSY
Eyes partially closed. Score increasing. Yellow warning zone. Driver monitored.
🚨
ALERT!
Critical drowsiness detected! Audio beep fires. Red border flashes. Immediate action required.

Architecture

CNN Architecture

Deep convolutional neural network with batch normalization and dropout regularization.

Input Layer — 80×80 Grayscale Image
↓
Conv2D(32) + BatchNorm + Conv2D(32) + MaxPool + Dropout(0.25)
↓
Conv2D(64) + BatchNorm + Conv2D(64) + MaxPool + Dropout(0.25)
↓
Conv2D(128) + BatchNorm + MaxPool + Dropout(0.25)
↓
Flatten Layer
↓
Dense(256) + Dropout(0.5) → Dense(128) + Dropout(0.3)
↓
Output — Dense(1) Sigmoid → Open / Closed
